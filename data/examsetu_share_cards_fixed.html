What was broken — and the fix
1
Massive headline font (100px+) overflows and overlaps the icon/right column — no max-width or overflow control
Fix: Use relative font sizes (36–42px), set explicit flex layout with justify-content:space-between so text and icon never collide
2
Bottom stat cards are separate floating elements — not part of the share card, causing misalignment when exported
Fix: Stat row is embedded inside the card using a dark semi-transparent background section — always travels with the card
3
Empty lower half of card — no content filling the space, looks unfinished and amateurish
Fix: Card uses flex-direction:column with flex:1 on body — content fills space. Stat row anchors the bottom third
4
gradient background causes visible banding and renders badly on social media compression (JPEG artifacts)
Fix: Solid dark brand colors — #2A1A0E / #0A1628 / #140A2E — clean on every platform, no artifacts
5
icon8 icons referenced as img src with no fallback — if CDN fails, broken image icon appears on the share card
Fix: Inline SVG for share cards (always works). Use icons8 for UI only, never for exported images
How to use Icons8 correctly in your app
For UI navigation & buttons
Use Icons8 CDN with img tag. Set explicit width/height always. Never let icon inherit parent font-size.
Best use
<img src="https://img.icons8.com/
  fluency/48/fire.png"
  width="20" height="20"
  alt="streak" />
For share cards & exports
Never use img tags in canvas-rendered or html2canvas exported cards. Icons8 URLs get blocked by CORS. Use inline SVG instead.
Critical rule
Pick the right style
Use one icon style throughout the app. Mixing Fluency + Color + Material looks inconsistent. Fluency works best for edtech UI — friendly but professional.
Design tip
Icons8 API token setup
With your premium token, append ?token=YOUR_TOKEN to each URL. Store it in .env, never hardcode in frontend JS.
Setup
// .env
ICONS8_TOKEN=your_token_here

// usage
const icon = `https://img.icons8.com/fluency/48/fire.png?token=${process.env.ICONS8_TOKEN}`
Copilot prompt — paste this into your system prompt
You are generating share cards for ExamSetu (UPTET prep app).
Each card is a 1:1 square, 1080x1080px equivalent.

LAYOUT RULES (follow strictly):
- card = flex column, dark solid bg (no gradients)
- header: app logo + name left, card-type badge right
- body: flex row — text left (max 60% width), icon right (70px circle)
- headline max font-size: 48px. Never let text overflow.
- stat-row: 3 equal columns, semi-transparent bg, bottom of card
- footer: caption text left, CTA button right

TYPOGRAPHY:
- eyebrow: 9-10px, 700 weight, 0.1em letter-spacing, brand color
- headline: 36-48px, 800 weight, white
- subhead: 13-14px, 600 weight, brand accent color
- desc: 10-11px, 400 weight, rgba(255,255,255,0.55)
- stat label: 8-9px, 700, uppercase, rgba(255,255,255,0.45)
- stat value: 16px, 700, rgba(255,255,255,0.95)

COLORS per card type:
- streak: bg #2A1A0E, accent #D85A30, light #F0997B
- achievement: bg #0A1628, accent #185FA5, light #85B7EB
- rank: bg #140A2E, accent #534AB7, light #AFA9EC

NEVER DO:
- gradient backgrounds
- font-size above 48px on headlines
- position:absolute for content (use flex)
- img tags for icons (use inline SVG)
- empty whitespace below stats row
- ALL CAPS for description text
